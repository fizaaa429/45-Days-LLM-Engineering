The ReAct agent:
one line that replaces your loop

Day 23 you wrote the tool-calling loop by hand. Day 24 you learned the graph engine. Today they fuse: create_agent(model, tools) builds a real agent that reasons, calls tools, and answers on its own.

Where we were

Day 23: you wrote the loop by hand

To make the model act, you wrote a while loop: ask the model, run any tool it asked for, feed the result back, repeat until it answers.

messages = [SystemMessage(...), HumanMessage(question)] for _ in range(MAX_STEPS): ai = model.invoke(messages) messages.append(ai) if not ai.tool_calls: return ai.content # ← plain answer, done for call in ai.tool_calls: result = TOOL_MAP[call["name"]].invoke(call["args"]) messages.append(ToolMessage(result, tool_call_id=call["id"]))

It worked — but every new agent meant re-writing this plumbing. There's a name for this pattern...

The pattern has a name

ReAct = Reason + Act

The model reasons about what it needs, acts by calling a tool, observes the result, and loops — until it can answer in plain text. Day 23's while loop was ReAct.

The reveal

All of that, in one line

from langchain.agents import create_agent agent = create_agent(model, tools) # ← the whole ReAct loop result = agent.invoke({"messages": [("human", "What is 240 + 360?")]}) answer = result["messages"][-1].content # "240 + 360 = 600."
What you just got back

create_agent returns a compiled LangGraph agent — literally yesterday's engine, pre-wired into a model → tools → model cycle. You stop writing loops and start using one.

Because you built it by hand first, this line is never magic — you know exactly what's inside.

What goes in

An agent = a model + a list of tools

@tool def multiply(a: int, b: int) -> int: """Multiply a and b.""" return a * b agent = create_agent( model, # calls tools [add, multiply], # the toolbox )
  • The tools are plain Day-23 @tool functions — nothing new.
  • The only change from Day 23: you no longer write the loop that calls them.
  • Input is {"messages": [("human", "…")]}; the answer is the last message.
Gotcha

The agent returns the whole conversation, not a string. Read result["messages"][-1].content — forgetting that is the #1 first-time mistake.

Inside the loop

Every step is a message

The agent's answer is a trail of messages — and it's exactly the shape of Day 23's loop:

1. HumanMessage "Multiply 6 by 7, then add 100." 2. AIMessage tool_calls=[multiply(6, 7)] ← Reason + Act 3. ToolMessage "42" ← Observe 4. AIMessage tool_calls=[add(42, 100)] ← Reason + Act 5. ToolMessage "142" ← Observe 6. AIMessage "6 x 7 = 42, and 42 + 100 = 142" ← Answer

Model → Tool → Model → Tool → Model. Same loop you hand-wrote — now generated for you.

Watch it live

Stream the steps as they happen

for chunk in agent.stream( {"messages": [("human", q)]}, stream_mode="updates"): for node, upd in chunk.items(): print(node, upd["messages"][-1]) # [model] wants: multiply(6, 7) # [tools] result: 42 # [model] wants: add(42, 100) # [tools] result: 142 # [model] "…= 142."

Two node names tell the whole story:

  • model — the LLM reasoning & acting.
  • tools — a tool being run.

That stream is what powers the "How I got this" tool-trace you'll show users in the mini-project.

Steer it

A system prompt + a toolbox it routes itself

agent = create_agent(model, [calculator, search_handbook, word_count], system_prompt="You are SoftBot. Always use a tool for maths or " "lookups — never guess. Keep answers short.")

🎭 System prompt

The agent's job description — persona + rules. "Never guess the maths, use the tool" is the classic one.

🧰 The model picks

Hand over many tools; ask a mixed question; it calls the right ones in order. You write zero routing. That self-routing is what makes it an agent.

So write good docstrings

A tool's docstring + argument names are the model's only instructions for when to use it. Vague docstring → wrong tool. Write it like you're briefing a new teammate.

Make it conversational

Memory: the same two knobs as Day 24

from langgraph.checkpoint.memory \ import MemorySaver agent = create_agent( model, tools, checkpointer=MemorySaver()) cfg = {"configurable": {"thread_id": "roll-42"}} agent.invoke({"messages": [("human", "Hi, I'm Aditi")]}, cfg) agent.invoke({"messages": [("human", "my name?")]}, cfg) # Aditi
  • Compile with a checkpointer → history is saved.
  • Pass a thread_id → same id remembers, new id starts clean.
  • Each turn send only the new message; the rest reloads.
Exactly Day 24

Identical to yesterday's chatbot — the only difference is this bot also has tools. One thread_id per user = many private chats from one agent.
